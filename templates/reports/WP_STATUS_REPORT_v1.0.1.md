# Work Package Status Report

Template ID: TPL-RPT-WP-STATUS-REPORT-v1.0.1
Template Version: v1.0.1
Artifact Type: WORK_PACKAGE_STATUS_REPORT
Output Intent: PDF-style narrative report
Target Scope: SINGLE_WP | SELECTED_WP_SET
Projection Only: true
Mutates WP Truth: false

---

## 1. Cover Page

Required fields:

- Report title
- Artifact ID
- Target scope
- WP ID(s)
- WP title(s)
- Area / system / equipment, if available
- Generated date/time
- Template ID/version
- Contract/action/version
- Source snapshot hash
- Projection-only statement

Required statement:

This report is a projection artifact generated from controlled WP snapshot data. It does not mutate WP/task truth and must not be treated as an approval, electronic signature, or QMS record unless separately approved through the appropriate external process.

---

## 2. Executive Summary

Required content:

- Overall status
- Progress summary
- Key accomplishments, if available
- Key blockers, if available
- Immediate next steps
- Readiness statement, if supported by available data

---

## 3. Work Package Overview

For SINGLE_WP, provide one overview block.

For SELECTED_WP_SET, provide a consolidated table and one subsection per WP.

Required fields:

- WP ID
- WP title
- WP lifecycle state/status
- WP type/scope
- Objective, if available
- Area/system/equipment, if available
- Owner/governance role, if available
- Source snapshot reference/hash

---

## 4. Task Status Summary

Required metrics:

- Total tasks
- Planned tasks
- In-progress tasks
- Completed tasks
- Open tasks
- Overdue tasks
- Blocked tasks, if status exists
- Tasks missing owner, if detectable
- Tasks missing dates, if detectable

---

## 5. Schedule Summary

Required content:

- Earliest start, if dates exist
- Latest finish, if dates exist
- Proposed vs committed distinction where applicable
- Delay/lateness summary
- Schedule risk statement
- Calendar basis, if schedule metrics exist

---

## 6. Risks / Issues / Exceptions

Required content when applicable:

- Overdue task list
- Missing required data
- Missing stamps
- Validation warnings
- Blocked artifact-generation reasons

If no risks/issues/exceptions are detected, state that none were detected from the available WP snapshot data.
